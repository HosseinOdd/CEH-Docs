## Common Ethical Hacking Terms

### Reconnaissance
The first phase of ethical hacking where information is gathered about the target system.

### Scanning
Identifying live hosts, open ports, and services running on the target network.

### Enumeration
Extracting detailed information about the target, such as usernames, network shares, and services.

### Gaining Access
Exploiting vulnerabilities to gain unauthorized access to systems or networks.

### Maintaining Access
Establishing persistent access to compromised systems using backdoors or rootkits.

### Covering Tracks
Removing evidence of the attack by clearing logs and hiding malicious activities.

## Key Terminology

- **Vulnerability**: A weakness in a system that can be exploited
- **Exploit**: Code or technique used to take advantage of a vulnerability
- **Payload**: The malicious code delivered after successful exploitation
- **Zero-Day**: A previously unknown vulnerability with no available patch
- **Social Engineering**: Manipulating people to divulge confidential information
- **Phishing**: Fraudulent attempts to obtain sensitive information via deceptive emails
- **Footprinting**: Collecting information about a target before an attack
- **Port Scanning**: Identifying open ports on target systems
- **Brute Force**: Trying multiple password combinations to gain access
- **Privilege Escalation**: Gaining higher-level permissions on a system

## Attack Vectors

- **Malware**: Malicious software including viruses, trojans, and ransomware
- **DDoS**: Distributed Denial of Service attack overwhelming system resources
- **Man-in-the-Middle (MitM)**: Intercepting communication between two parties
- **SQL Injection**: Exploiting database vulnerabilities through malicious queries
- **Cross-Site Scripting (XSS)**: Injecting malicious scripts into web applications
