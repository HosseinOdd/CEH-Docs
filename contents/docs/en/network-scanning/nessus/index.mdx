## What is Nessus?

Nessus is one of the most popular vulnerability scanners developed by Tenable. It provides comprehensive vulnerability scanning, configuration auditing, and compliance checking for networks and systems.

## Key Features

- Comprehensive vulnerability database
- Network scanning and discovery
- Web application scanning
- Configuration auditing
- Compliance checking
- Patch management
- Credentialed scanning
- Extensive reporting

## Nessus Editions

### Nessus Essentials (Free)
- Limited to 16 IP addresses
- Basic vulnerability scanning
- Good for home labs and learning

### Nessus Professional
- Unlimited IP addresses
- Advanced scanning features
- Live results
- Unlimited scans

### Nessus Expert
- Includes Professional features
- Cloud infrastructure scanning
- Container security

## Installation

### Linux
```bash
# Download from https://www.tenable.com/downloads/nessus
sudo dpkg -i Nessus-<version>-debian6_amd64.deb
sudo systemctl start nessusd
```

### Windows
Download installer and follow GUI setup.

### Access Web Interface
```
https://localhost:8834
```

## Basic Usage

### 1. Initial Setup
- Create admin account
- Get activation code
- Download plugins

### 2. Creating a Scan
- Click "New Scan"
- Choose scan template
- Configure targets
- Set scan settings
- Launch scan

### 3. Scan Templates

**Basic Network Scan**
- General purpose scanning
- Identifies live hosts and open ports
- Detects services and vulnerabilities

**Web Application Scan**
- Specifically for web applications
- Tests for common web vulnerabilities
- OWASP Top 10 coverage

**Advanced Scan**
- Fully customizable
- Granular control over plugins
- Advanced options

**Credentialed Scan**
- Uses credentials to access systems
- More accurate results
- Can detect missing patches

## Understanding Results

### Vulnerability Severity

**Critical**
- Severe vulnerabilities
- Immediate action required
- Remote code execution likely

**High**
- Significant vulnerabilities
- Should be addressed quickly
- Potential for compromise

**Medium**
- Moderate risk
- Should be addressed
- May aid in attacks

**Low**
- Minor issues
- Informational
- Low risk

**Info**
- Informational only
- No direct vulnerability
- Useful for inventory

## Best Practices

### Before Scanning
1. Get proper authorization
2. Notify network team
3. Schedule during maintenance window
4. Test on small subnet first

### During Scanning
1. Monitor scan progress
2. Watch for network impact
3. Adjust scan speed if needed
4. Pause if issues occur

### After Scanning
1. Review all findings
2. Verify critical vulnerabilities
3. Prioritize remediation
4. Document actions taken

## Troubleshooting

### Common Issues

**Plugins Not Updating:**
- Check internet connection
- Verify activation code
- Restart Nessus service

**Scan Hanging:**
- Reduce scan speed
- Check target connectivity
- Review firewall rules

**False Positives:**
- Enable safe checks
- Use credentials
- Verify manually
