## هدر ایمیل چیست؟

هدر ایمیل حاوی متادیتا درباره سفر ایمیل از فرستنده به گیرنده است. این شامل اطلاعات مسیریابی، برچسب‌های زمانی، نتایج احراز هویت و جزئیات فنی است که معمولاً از کاربران مخفی است.

## اجزای هدرهای ایمیل

### فیلد From
آدرس ایمیل فرستنده (می‌تواند جعلی باشد).

### فیلد To
آدرس ایمیل گیرنده.

### Subject
خط موضوع ایمیل.

### Date
برچسب زمانی ارسال ایمیل.

### Return-Path
آدرس ایمیلی که باید پیام‌های برگشتی به آن ارسال شود.

### Received
زنجیره سرورهای ایمیل که ایمیل را مدیریت کردند و مسیر کامل را نشان می‌دهند.

### Message-ID
شناسه منحصر به فرد برای ایمیل.

### MIME-Version
نسخه Multipurpose Internet Mail Extensions.

### Content-Type
فرمت محتوای ایمیل (متن، HTML و غیره).

### X-Headers
هدرهای سفارشی اضافه شده توسط سرورها یا کلاینت‌های ایمیل.

## فیلدهای مهم هدر برای تحلیل

### هدرهای Received
مسیری که ایمیل از طریق سرورهای مختلف ایمیل طی کرده را نشان می‌دهد:
```
Received: from server1.example.com
    by server2.example.com
    with SMTP id xyz123
    for <recipient@example.com>;
    Mon, 1 Jan 2024 12:00:00 +0000
```

### نتایج احراز هویت
- **SPF**: اعتبارسنجی Sender Policy Framework
- **DKIM**: امضای DomainKeys Identified Mail
- **DMARC**: احراز هویت پیام مبتنی بر دامنه

### X-Originating-IP
آدرس IP اصلی فرستنده.

### X-Mailer
نرم‌افزار کلاینت ایمیل استفاده شده برای ارسال پیام.

## نحوه مشاهده هدرهای ایمیل

### Gmail
1. ایمیل را باز کنید
2. روی منوی سه نقطه کلیک کنید
3. "Show original" را انتخاب کنید

### Outlook
1. ایمیل را باز کنید
2. File → Properties
3. هدرها را در جعبه "Internet headers" مشاهده کنید

### Apple Mail
1. ایمیل را باز کنید
2. View → Message → Raw Source

## فرآیند تحلیل هدر ایمیل

### 1. شناسایی فرستنده
- آدرس "From" را بررسی کنید
- در مقابل Return-Path تأیید کنید
- به دنبال X-Originating-IP بگردید

### 2. ردیابی مسیر
- هدرهای Received را از پایین به بالا بخوانید
- هر جهش سرور ایمیل را شناسایی کنید
- برچسب‌های زمانی هر جهش را یادداشت کنید

### 3. بررسی احراز هویت
- نتایج SPF را تأیید کنید
- امضای DKIM را بررسی کنید
- خط‌مشی DMARC را مرور کنید

### 4. تشخیص ناهنجاری‌ها
- ناسازگاری منطقه زمانی
- آدرس‌های IP مشکوک
- دامنه‌های ناهماهنگ
- احراز هویت گمشده

## تهدیدات رایج ایمیل

### جعل ایمیل (Email Spoofing)
جعل آدرس فرستنده برای ظاهر شدن از منبع معتبر.

### فیشینگ
ایمیل‌های کلاهبردارانه که تلاش می‌کنند اعتبارنامه یا اطلاعات را بدزدند.

### سازش ایمیل تجاری (BEC)
جعل هویت مدیران اجرایی برای تأیید تراکنش‌های کلاهبردارانه.

### توزیع بدافزار
ایمیل‌های حاوی پیوست‌ها یا لینک‌های مخرب.

## ابزارهای تحلیل هدر ایمیل

### ابزارهای آنلاین

**MXToolbox Email Header Analyzer**
پارس و تحلیل هدرهای ایمیل با نمایش بصری.

**Google Admin Toolbox Message Header**
ابزار گوگل برای تحلیل مسیریابی و احراز هویت ایمیل.

**Mail Header Analyzer**
تحلیل دقیق هدرهای ایمیل با امتیاز هرزنامه.

### ابزارهای خط فرمان

**emlAnalyzer**
ابزار Python برای پارس و تحلیل فایل‌های EML.

**Email Header Analyzer**
ابزار متن‌باز برای تحلیل قانونی ایمیل.

## پرچم‌های قرمز در هدرهای ایمیل

- چندین خط "Received" از سرورهای مشکوک
- دامنه‌های ناهماهنگ فرستنده
- شکست بررسی‌های SPF/DKIM/DMARC
- مکان‌های غیرعادی X-Originating-IP
- Message-ID گمشده یا مشکوک
- اختلافات زمانی
- مقادیر مشکوک X-Mailer

## موارد استفاده تحقیقاتی

### تحقیق هرزنامه
شناسایی منبع ایمیل‌های هرزنامه و گزارش به ISPها.

### تشخیص فیشینگ
تأیید اصالت ایمیل‌های مشکوک.

### جعل ایمیل
تشخیص ایمیل‌های جعلی یا کلاهبردارانه.

### شواهد قانونی
حفظ هدرهای ایمیل به عنوان شواهد قانونی.

### عیب‌یابی
تشخیص مشکلات تحویل ایمیل.

## بهترین روش‌ها

- همیشه هدرها را برای ایمیل‌های مشکوک بررسی کنید
- نتایج احراز هویت فرستنده را تأیید کنید
- اطلاعات هدر را برای تحقیقات مستند کنید
- از چندین ابزار تحلیل برای تأیید استفاده کنید
- تلاش‌های فیشینگ را به مقامات مربوطه گزارش دهید
- هرگز تنها به فیلد "From" اعتماد نکنید
- به دنبال رمزگذاری SSL/TLS در حین انتقال باشید
