## مهندسی اجتماعی چیست؟

مهندسی اجتماعی دستکاری روانشناختی افراد برای انجام اقدامات یا افشای اطلاعات محرمانه است. این روش به جای آسیب‌پذیری‌های فنی، روانشناسی انسان را مورد سوء استفاده قرار می‌دهد.

## انواع مهندسی اجتماعی

### مهندسی اجتماعی مبتنی بر انسان

**جعل هویت (Impersonation)**
وانمود کردن به شخص دیگری برای جلب اعتماد و دسترسی به اطلاعات.

**استراق سمع (Eavesdropping)**
گوش دادن به مکالمات خصوصی برای جمع‌آوری اطلاعات حساس.

**نگاه کردن از روی شانه (Shoulder Surfing)**
مشاهده صفحه نمایش یا صفحه کلید شخصی برای به دست آوردن اطلاعات.

**جستجو در زباله (Dumpster Diving)**
جستجو در زباله برای یافتن اسناد یا اطلاعات محرمانه.

**دنبال کردن (Tailgating)**
دنبال کردن پرسنل مجاز به مناطق محدود.

### مهندسی اجتماعی مبتنی بر کامپیوتر

**فیشینگ (Phishing)**
ارسال ایمیل‌های جعلی که از منابع معتبر به نظر می‌رسند برای سرقت داده‌های حساس.

**فیشینگ هدفمند (Spear Phishing)**
حملات فیشینگ هدفمند به افراد یا سازمان‌های خاص.

**فیشینگ صوتی (Vishing)**
استفاده از تماس تلفنی برای فریب قربانیان به افشای اطلاعات.

**فیشینگ پیامکی (Smishing)**
استفاده از پیام‌های متنی برای فریب قربانیان.

**طعمه‌گذاری (Baiting)**
ارائه چیزی جذاب برای کشیدن قربانیان به دام.

**بهانه‌سازی (Pretexting)**
ایجاد سناریوی ساختگی برای به دست آوردن اطلاعات.

## تکنیک‌های رایج مهندسی اجتماعی

### اقتدار
وانمود کردن به داشتن قدرت برای فرمان‌برداری.

### فوریت
ایجاد احساس فشار زمانی برای دور زدن تفکر منطقی.

### ارعاب
استفاده از تهدید یا رفتار تهاجمی برای وادار کردن به اطاعت.

### اعتماد
ایجاد رابطه و سوء استفاده از اعتماد قربانی.

### کمیابی
ایجاد احساس محدود یا نادر بودن فرصت.

## چرخه حمله مهندسی اجتماعی

1. **تحقیق**: جمع‌آوری اطلاعات درباره هدف
2. **قلاب**: برقراری تماس و ایجاد اعتماد
3. **اجرا**: اجرای سناریوی حمله
4. **خروج**: ترک بدون ایجاد سوء ظن

## علائم هشدار

- درخواست‌های غیرمعمول برای اطلاعات حساس
- فشار برای اقدام سریع
- درخواست‌ها برای دور زدن رویه‌های امنیتی
- پیشنهادات خیلی خوب برای باور کردن
- تماس‌های تأیید نشده یا مشکوک

## پیشگیری و دفاع

### برای سازمان‌ها
- آموزش آگاهی امنیتی
- سیاست‌های امنیتی واضح
- رویه‌های تأیید
- سیستم گزارش‌دهی رویدادها
- تمرین‌های امنیتی منظم

### برای افراد
- هویت را قبل از اشتراک اطلاعات تأیید کنید
- نسبت به درخواست‌های ناخواسته بدبین باشید
- هرگز رمز عبور یا داده‌های حساس را به اشتراک نگذارید
- قبل از کلیک روی لینک یا پیوست فکر کنید
- فعالیت‌های مشکوک را گزارش دهید

## نمونه‌های واقعی

- کلاهبرداری CEO: جعل هویت مدیران اجرایی برای تأیید انتقال‌های کلاهبردارانه
- کلاهبرداری پشتیبانی فنی: وانمود به پشتیبانی IT برای دسترسی به سیستم
- کلاهبرداری عاشقانه: ایجاد روابط جعلی برای سوء استفاده مالی از قربانیان
- کلاهبرداری لاتاری: ادعای برنده شدن جوایز برای استخراج هزینه یا اطلاعات
