## Nessus چیست؟

Nessus یکی از محبوب‌ترین اسکنرهای آسیب‌پذیری است که توسط Tenable توسعه یافته است. این ابزار اسکن جامع آسیب‌پذیری، ممیزی پیکربندی و بررسی انطباق را برای شبکه‌ها و سیستم‌ها ارائه می‌دهد.

## ویژگی‌های کلیدی

- پایگاه داده جامع آسیب‌پذیری
- اسکن و کشف شبکه
- اسکن برنامه‌های وب
- ممیزی پیکربندی
- بررسی انطباق
- مدیریت وصله
- اسکن با اعتبارنامه
- گزارش‌دهی گسترده

## نسخه‌های Nessus

### Nessus Essentials (رایگان)
- محدود به 16 آدرس IP
- اسکن پایه آسیب‌پذیری
- مناسب برای آزمایشگاه‌های خانگی و یادگیری

### Nessus Professional
- آدرس‌های IP نامحدود
- ویژگی‌های اسکن پیشرفته
- نتایج زنده
- اسکن‌های نامحدود

### Nessus Expert
- شامل ویژگی‌های Professional
- اسکن زیرساخت ابری
- امنیت کانتینر

## نصب

### Linux
```bash
# دانلود از https://www.tenable.com/downloads/nessus
sudo dpkg -i Nessus-<version>-debian6_amd64.deb
sudo systemctl start nessusd
```

### Windows
نصب‌کننده را دانلود کرده و راه‌اندازی GUI را دنبال کنید.

### دسترسی به رابط وب
```
https://localhost:8834
```

## استفاده پایه

### 1. راه‌اندازی اولیه
- ایجاد حساب مدیر
- دریافت کد فعال‌سازی
- دانلود افزونه‌ها

### 2. ایجاد اسکن
- کلیک بر "New Scan"
- انتخاب قالب اسکن
- پیکربندی اهداف
- تنظیم تنظیمات اسکن
- راه‌اندازی اسکن

### 3. قالب‌های اسکن

**اسکن پایه شبکه**
- اسکن عمومی
- شناسایی هاست‌های زنده و پورت‌های باز
- تشخیص سرویس‌ها و آسیب‌پذیری‌ها

**اسکن برنامه وب**
- مخصوص برنامه‌های وب
- تست آسیب‌پذیری‌های رایج وب
- پوشش OWASP Top 10

**اسکن پیشرفته**
- کاملاً قابل سفارشی‌سازی
- کنترل دقیق بر افزونه‌ها
- گزینه‌های پیشرفته

**اسکن با اعتبارنامه**
- از اعتبارنامه برای دسترسی به سیستم‌ها استفاده می‌کند
- نتایج دقیق‌تر
- می‌تواند وصله‌های گمشده را تشخیص دهد

## درک نتایج

### شدت آسیب‌پذیری

**بحرانی (Critical)**
- آسیب‌پذیری‌های شدید
- نیاز به اقدام فوری
- احتمال اجرای کد از راه دور

**بالا (High)**
- آسیب‌پذیری‌های قابل توجه
- باید سریع رسیدگی شود
- پتانسیل برای سازش

**متوسط (Medium)**
- ریسک متوسط
- باید رسیدگی شود
- ممکن است در حملات کمک کند

**پایین (Low)**
- مسائل جزئی
- اطلاعاتی
- ریسک پایین

**اطلاعات (Info)**
- فقط اطلاعاتی
- بدون آسیب‌پذیری مستقیم
- مفید برای فهرست‌برداری

## بهترین روش‌ها

### قبل از اسکن
1. کسب مجوز مناسب
2. اطلاع به تیم شبکه
3. زمان‌بندی در طول پنجره نگهداری
4. تست در زیرشبکه کوچک ابتدا

### در طول اسکن
1. نظارت بر پیشرفت اسکن
2. تماشای تأثیر شبکه
3. تنظیم سرعت اسکن در صورت نیاز
4. توقف در صورت بروز مشکل

### بعد از اسکن
1. بررسی تمام یافته‌ها
2. تأیید آسیب‌پذیری‌های بحرانی
3. اولویت‌بندی رفع
4. مستندسازی اقدامات انجام شده
