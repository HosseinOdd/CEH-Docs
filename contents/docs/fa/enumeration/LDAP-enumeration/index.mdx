## معرفی

سرشماری LDAP فرآیند جمع‌آوری اطلاعات دقیق از سرویس‌های دایرکتوری است که از پروتکل Lightweight Directory Access Protocol استفاده می‌کنند. LDAP به‌طور گسترده در محیط‌های سازمانی برای احراز هویت متمرکز، کنترل دسترسی و مدیریت دایرکتوری به‌کار می‌رود.

پیکربندی نادرست سرویس‌های LDAP می‌تواند اطلاعات بسیار حساسی درباره کاربران و ساختار سازمانی افشا کند.

## LDAP چیست

LDAP یا Lightweight Directory Access Protocol یک پروتکل لایه کاربردی است که برای دسترسی و مدیریت اطلاعات دایرکتوری طراحی شده است. این پروتکل در فناوری‌هایی مانند Active Directory، OpenLDAP و سایر سیستم‌های مبتنی بر دایرکتوری استفاده می‌شود.

LDAP معمولاً روی پورت‌های TCP 389 (LDAP) و 636 (LDAPS) اجرا می‌شود.

## اطلاعات قابل استخراج از LDAP

در سرشماری LDAP، مهاجم یا تست‌کننده نفوذ می‌تواند به اطلاعات زیر دست پیدا کند:
- حساب‌های کاربری و نام‌های کاربری
- گروه‌ها و عضویت کاربران در آن‌ها
- واحدهای سازمانی (Organizational Units)
- ساختار دامنه و دایرکتوری
- آدرس‌های ایمیل و اطلاعات تماس
- سیاست‌های رمز عبور و ویژگی‌های حساب‌ها

در برخی موارد، LDAP امکان اتصال ناشناس (Anonymous Bind) را فراهم می‌کند که Enumeration را بدون احراز هویت ممکن می‌سازد.

## تأثیر امنیتی

سرشماری موفق LDAP می‌تواند منجر به:
- انجام حملات هدفمند روی حساب‌های کاربری
- افشای ساختار داخلی سازمان
- افزایش سطح دسترسی و حرکت جانبی در شبکه
- تسهیل حملات Active Directory شود

از دیدگاه دفاعی، سرویس‌های LDAP ناامن یکی از بزرگ‌ترین ریسک‌های امنیتی در شبکه‌های سازمانی هستند.

## جایگاه در هک اخلاقی و CEH

در چارچوب CEH، سرشماری LDAP به‌عنوان یکی از تکنیک‌های پراثر در جمع‌آوری اطلاعات آموزش داده می‌شود. هکرهای اخلاقی باید کوئری‌های LDAP، ساختار دایرکتوری و کنترل‌های دسترسی را به‌خوبی درک کنند تا امنیت سرویس‌های دایرکتوری را به‌درستی ارزیابی کنند.
