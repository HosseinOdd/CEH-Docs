## معرفی

سرشماری DNS فرآیند جمع‌آوری اطلاعات از سامانه نام دامنه (Domain Name System) است. DNS نقش حیاتی در تبدیل نام دامنه‌ها به آدرس‌های IP دارد و در صورت پیکربندی نادرست می‌تواند اطلاعات ارزشمندی از زیرساخت داخلی و خارجی یک سازمان افشا کند.

این نوع Enumeration معمولاً در مراحل اولیه حمله انجام می‌شود تا نقشه دقیقی از شبکه هدف ایجاد گردد.

## DNS چیست

DNS یا Domain Name System یک سیستم نام‌گذاری سلسله‌مراتبی و توزیع‌شده است که نام‌های قابل خواندن توسط انسان را به آدرس‌های IP تبدیل می‌کند. علاوه بر این، DNS اطلاعاتی درباره سرورهای ایمیل، Name Serverها و سایر سرویس‌ها ارائه می‌دهد.

از آنجایی که DNS معمولاً به‌صورت عمومی در دسترس است، یکی از اهداف اصلی جمع‌آوری اطلاعات محسوب می‌شود.

## اطلاعات قابل استخراج از DNS

در سرشماری DNS، مهاجم یا تست‌کننده نفوذ می‌تواند به اطلاعات زیر دست پیدا کند:
- نام دامنه و زیر دامنه‌ها
- آدرس‌های IP سرورها و میزبان‌ها
- Name Serverها (رکوردهای NS)
- Mail Serverها (رکوردهای MX)
- اطلاعات سرویس‌ها (رکوردهای TXT و SRV)
- محدوده‌های شبکه و نام میزبان‌های داخلی

در برخی موارد، پیکربندی نادرست DNS امکان Zone Transfer را فراهم کرده و کل رکوردهای دامنه را افشا می‌کند.

## تأثیر امنیتی

سرشماری موفق DNS می‌تواند منجر به:
- افشای ساختار داخلی شبکه
- شناسایی سرورها و سرویس‌های حیاتی
- انجام حملات فیشینگ یا ایمیل هدفمند
- پشتیبانی از مراحل بعدی اسکن و نفوذ شود

از دیدگاه دفاعی، پیکربندی نادرست DNS یکی از رایج‌ترین منابع نشت اطلاعات است.

## جایگاه در هک اخلاقی و CEH

در دوره CEH، سرشماری DNS به‌عنوان یکی از تکنیک‌های پایه شناسایی و Enumeration آموزش داده می‌شود. هکرهای اخلاقی باید رفتار DNS را به‌خوبی درک کنند تا رکوردهای افشاشده، خطاهای پیکربندی و ضعف‌های زیرساخت دامنه را شناسایی کنند.
