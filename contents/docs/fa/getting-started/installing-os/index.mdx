# نصب Metasploitable


Metasploitable یک سیستم‌عامل عمداً ناامن است که برای تمرین تست نفوذ و یادگیری آسیب‌پذیری‌ها طراحی شده است. معمولاً آن را روی ماشین مجازی (VM) اجرا می‌کنیم تا در محیطی ایزوله تمرین کنیم.


> ⚠️ هشدار: Metasploitable به صورت عمدی دارای آسیب‌پذیری‌های زیاد است. آن را **هرگز** مستقیماً به شبکه‌ی تولید یا اینترنت متصل نکنید. از شبکه‌ی ایزوله (Host-Only) یا شبکه‌ی آزمایش (NAT) استفاده کنید.


---


## فایل‌های استخراج‌شده پس از unzip


| فایل | توضیح |
|---|---|
| `metasploitable.vmdk` | دیسک مجازی اصلی — **ضروری** |
| `metasploitable.vmx` | تنظیمات VMware (برای ارجاع) |
| `metasploitable.vmsd` | متادیتای Snapshot (اختیاری) |
| `metasploitable.nvram` | حالت NVRAM/BIOS (معمولاً لازم نیست) |
| `metasploitable.vmxf` | متادیتای اضافی VMware (غیرضروری) |


> خلاصه: برای اجرای در **VirtualBox** مهم‌ترین فایل `metasploitable.vmdk` است. بقیه فایل‌ها مربوط به VMware هستند و می‌توانید تنظیمات معادل را در VirtualBox دستی بسازید.


---


## نصب Metasploitable روی VirtualBox


### روش ۱ — استفاده از VMDK مستقیم (ساده)


1. VirtualBox را باز کنید.
2. از منو روی **New** کلیک کنید.
- نام: `Metasploitable`
- Type: `Linux`
- Version: اگر پرسیده شد، `Other Linux (32-bit)` یا نزدیک‌ترین مورد را انتخاب کنید.
3. مقدار RAM: حدود **512 MB** تا **1 GB** (بستگی به نسخه).
4. When asked for a hard disk: انتخاب کنید **Use an existing virtual hard disk file** و `metasploitable.vmdk` را انتخاب کنید.
5. ایجاد VM را تمام کنید و سپس تنظیمات ماشین را باز کنید:
- **System → Motherboard**: Boot Order را بررسی کنید (Hard Disk فعال باشد).
- **System → Processor**: حداقل 1 CPU انتخاب کنید.
- **Network**: توصیه می‌شود از **NAT** یا **Host-Only Adapter** استفاده کنید. برای آزمایش‌های امن و ایزوله، Host-Only یا Internal Network را در نظر بگیرید.
6. ماشین را استارت کنید. احتمالاً پس از چند لحظه وارد صفحه‌ی لاگین Metasploitable خواهید شد (نام کاربری/رمز معمولاً `msfadmin:msfadmin` برای نسخه‌های معروف است، اما قبل از استفاده چک کنید).


### روش ۲ — تبدیل VMDK به VDI (اختیاری، توصیه‌شده برای سازگاری بهتر)


اگر ترجیح می‌دهید از فرمت بومی VirtualBox (VDI) استفاده کنید:


- در محیطی که VirtualBox نصب است، از ترمینال/Command Prompt:


```bash
VBoxManage clonehd metasploitable.vmdk metasploitable.vdi --format VDI
```


- سپس هنگام ساخت VM، از گزینه **Use an existing virtual hard disk file** فایل `metasploitable.vdi` را انتخاب کنید.


### تنظیمات شبکه پیشنهادی برای تمرین (امن)


- **Host-Only**: فقط میزبان (کامپیوتر شما) و VM به هم دسترسی دارند — مناسب برای تمرین ایزوله.
- **NAT + Port Forwarding**: اگر لازم است از میزبان به سرویس‌های داخل VM دسترسی پیدا کنید (مثل SSH) می‌توانید پورت فورواردینگ فعال کنید.
- **هرگز** شبکه‌ی Bridged را در شبکه‌ی عمومی یا تولید فعال نکنید مگر کاملاً آگاه به پیامدها باشید.


---


## نکات کاربردی و امنیّت


- پس از نصب، Metasploitable را از اینترنت جدا نگه دارید.
- از snapshot قبل از تغییرات مهم استفاده کنید تا بتوانید به حالت اولیه بازگردید.
- برای تمرین همزمان با ابزارهای تست نفوذ (مثل Kali یا ماشین‌های دیگر)، از شبکه‌ی Host-Only یا Internal استفاده کنید.
- اطلاعات ورود پیش‌فرض را تغییر ندهید مگر بدانید چه می‌کنید — چون بسیاری از تمرین‌ها روی حساب‌ها/پلاگین‌های پیش‌فرض بنا شده‌اند.
